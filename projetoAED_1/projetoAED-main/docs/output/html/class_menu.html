Detailed Description

Class for all the menus.

Stores all the menus that can be accessed throughout the tool's runtime, and the data structures that hold the information provided in the files.

See also
Storage.hpp

Constructor & Destructor Documentation

◆ Menu()

Menu::Menu ( )

Constructor of menus' interface.

Initializes the interface to run the menus. Processes all the files into the respective data structures. Initializes the menuText member variable with a placeholder text.

See also
AllTurmas::AllTurmas()
AllTurmas::setSchedule()
AllStudents::AllStudents()
Note
Time complexity: O(k) + O(l*log p) + O(m*n*log p).
  • where k,l,m are number of lines in the files classes_per_uc.csv, classes.csv, students_classes.csv, respectively;
  • n is the number of Curricular Units the student is enrolled in, and p is the number of Turma objects in this->turmas;
    Notice: n is never higher than 7, because a student cannot be unrolled in more than 7 CUs at the same time.

◆ chooser()

int Menu::chooser ( bool  valid,
int  previous,
int  input_num = -1,
const string &  input_codes = "" 
)

Function that calls every other function according.

Chooses the next menu (printMenu() functions) and the functions needed to access/process the data, based on the previous menu's signature and the input provided.
Uses a switch-case for every possible menu option (integer inputs), as well as string inputs. Inside each case, another switch-case takes the previous menu's integer signature into consideration, to decide which menu to print next and which functions to call.

Parameters
validValidity of the input, decided by validDetect() function.
previousPrevious menu's integer signature (every menu has a different one).
input_numUser's valid int input, defaults to -1 when the user inputs a valid string.
input_codesUser's valid string input, split into arguments if needed that will be passed.
Returns
The printed menu's integer signature. It will be evaluated in the next loop, together with the input provided, to decide which menu to print next.
See also
validDetect()
Note
Time complexity: Depends on the input, given the multitude of options and functions that can be called. Refer to the time complexity of the other functions in this tool.

◆ printInvalidInput()

int Menu::printInvalidInput ( )

ERROR: INVALID INPUT Menu.

This function returns an integer signature that is significantly higher than the other ones. It was a choice made to facilitate the identification of the Error Menu and the ones related to it.
When the inputted number is not one of the displayed options, the Error Menu is printed. This menu's option to "Write a new input" raises the need to, in the next iteration of loop, save not only the previous/latest menu (this error menu), but also the menu before that, where the invalid input came from. Check the chooser() function to further understand the implementation.

Returns
This menu's integer signature (1000).
See also
chooser()
Note
Time complexity: O(1).

◆ validDetect()

int Menu::validDetect ( int  previous,
const string &  input_choice 
)

Detects if inputs are valid.

Input string is made up of an int: converted to int; evaluated if valid menu option or not.
Input string is made up of characters: function splits the string into arguments if needed; determines if inputted arguments are following the expected syntax with regular expressions.

Parameters
previousPrevious menu's integer signature (every menu has a different one).
input_choiceUser input: can be made up of only an int value (menu option); or string of characters (specific input).
Returns
0 if invalid input; 1 if valid string; 2 if valid int.
Note
Time complexity: O(n) for both types of input: when it is turned into a digit and when it is treated as a string.
