Detailed Description

Class that stores and manipulates all the Request objects as a whole.

Uses a vector to facilitate the manipulation of the stored data.

Auxiliary function that evaluates validity of Class Change Requests.

Parameters
requestRequest that is being checked.
all_students,all_turmasUsed to access the necessary data.
Returns
-2 if the new class has maximum occupation;
-1 if the balance between the old class and the new class has been;
0 if there is schedule conflict between the new class and the classes Student is already enrolled in;
1 otherwise -> valid Request;
See also
AllTurmas::classBalance()
ScheduleSlot::scheduleConflict()
processPending()
undoOneProcessed()
Note
Time complexity: O(log n + log p + q*(log n + r*s)).
  • where n is the number of Turma objects in this->turmas of the all_turmas object, p is the number of Student objects in this->students of the all_students object, q is the number of Curricular Units the student has enrolled in, r is the number of ScheduleSlot objects in one of the classes the student is enrolled in, s is the number of ScheduleSlot objects in the class the student wants to enroll in.

Auxiliary function that evaluates validity of CU Enroll Requests.

Parameters
requestRequest that is being checked.
all_students,all_turmasUsed to access the necessary data.
Returns
"-2" if the student is already enrolled in that class;
"-1" if the student is already enrolled in 7 CU;
"0" if all the classes of the desired CU are full or if there is a schedule conflict between the available classes and the classes Student is already enrolled in;
A string with the class's code otherwise -> valid Request;
See also
AllTurmas::classBalance()
processPending()
Note
Time complexity: O(n*(p*(log n + q) + r*s)).
  • where n is the number Turma objects in all_turmas.getTurmas(), p is the number of Curricular Units the student has enrolled in, q is the number of ScheduleSlot objects in one of the student's classes, r is the number of ScheduleSlot objects among all of the student's classes, and s is the number of ScheduleSlot objects in the class the student wants to enroll in.

void AllRequests::processPending ( AllStudents &  all_students,
AllTurmas &  all_turmas 
)

Function that processes the pending Requests and prints a message reflecting their state.

Evaluates if the requests are valid, using the checkRequestUcE() and checkRequestCc() functions, and if they are, makes the necessary changes in the data structures, through the given parameters.

Parameters
all_students,all_turmasUsed to manipulate, as needed, the students' and classes' data, in order to store the changes.
See also
AllStudents::getOneStudent()
AllTurmas::getOneTurma()
checkRequestUcE()
checkRequestCc()
Note
Time complexity:
>> O(n) when processing a CU enroll, where n is the number of requests in the pending queue;
>> O(n*(p+q)) when processing a CU remove, where p is the number of Student objects in the Student's old class, and q the number of classes the student has already enrolled in;
>> O(n*r) when processing a Class Change, where r is the number of Student objects in the Student's old class;

Function that writes the changes back into the files.

The only file that changes through all the manipulation options is students_classes.csv.
Through the given parameter, the function accesses the all_students vector (where all the students are stored) and prints the necessary information to the .csv file.

Parameters

Note
